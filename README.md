### 🎯 Назначение навыка

**Autonomous Wave Orchestrator** — это системный промпт для LLM-агента, который выступает в роли Главного Архитектора и Оркестратора. Он предназначен для автоматической декомпозиции сложных задач разработки ПО в структурированный, отказоустойчивый план выполнения.

### ⚙️ Как это работает

Агент получает на вход **только описание задачи и контекст** (без готового плана) и самостоятельно выполняет полный цикл планирования:

1.  **Бриф:** Формализует проблему, решение и ограничения.
2.  **Черновой план:** Набрасывает первичную последовательность шагов.
3.  **Адская самокритика:** Прогоняет план через 7 итераций анализа под разными линзами (безопасность, интеграция, отказы, дрейф контекста и др.), выявляя и устраняя уязвимости.
4.  **Волновая архитектура:** Преобразует доработанный план в 5–6 последовательных волн, где каждая волна выполняется 4 субагентами (Core, Integration, Infra, QA/Docs).
5.  **Протокол оркестрации:** Генерирует строгий алгоритм собственного управления: pre-wave проверки, сбор артефактов, сжатие контекста и матрицу обработки ошибок.

### 🔑 Ключевые особенности

-   **Главный агент не пишет код** — только координирует, валидирует и передаёт контекст.
-   **Жёсткая структура задач:** Каждая задача субагента имеет чёткие Input / Action / Output / Acceptance Criteria.
-   **QA как равноправный участник:** Субагент тестирования и документации встроен в каждую волну и имеет право вето на завершение этапа.
-   **Защита от дрейфа контекста:** Между волнами передаётся только сжатая «Сводка состояния», а не полная история чата.
-   **Отказоустойчивость:** Встроенные gate-checks перед каждой волной и матрица обработки ошибок трёх уровней.

### 📥 Входные данные

Агент принимает **только задачу + контекст** (стек, ограничения, особые требования). Готовый план давать не нужно — агент сам проектирует архитектуру и проходит цикл самокритики, чтобы выдать исполняемый волновой план.
